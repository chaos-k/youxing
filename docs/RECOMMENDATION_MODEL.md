# 推荐模型与知识库设计

## 数据输入

推荐所需信息分成四类：

| 类型 | 示例字段 | 用途 |
| --- | --- | --- |
| 用户基础 | 性别表达、年龄阶段 | 决定题目模板、成熟度和风格边界 |
| 面部发型 | 脸型、五官轮廓、颈长、发量、发质 | 生成发型和发色方向 |
| 身形比例 | 肩线、腰线、体型、腿型、身高区间 | 生成上装、下装、外套和鞋履版型 |
| 场景目标 | 通勤、校园、亲子、银发、正式度、季节、目标效果 | 决定穿搭组合和优先级 |

## 自适应题目模板

题目不是固定清单，而是由入口和年龄人群共同决定：

```text
入口 + 年龄阶段 -> 题目模板 -> 推荐知识库 -> 报告
```

示例：

- 宝宝完整诊断：性别、年龄、儿童场景、季节、亲肤/保暖/穿脱优先级、肤色。
- 大学生完整诊断：性别、年龄、脸型、肩线、体型、肤色、发量、校园场景、季节、目标效果。
- 成年人身形版型：性别、年龄、身高、肩线、体型、腰线、腿型、场景、正式度、目标效果。
- 发型定位：性别、年龄、脸型、颈长、发量、发质、肤色、风格偏好。

## 知识库结构

每条推荐知识可抽象成：

```js
{
  id: 'body_a_001',
  category: 'bottom',
  conditions: {
    bodyType: ['a']
  },
  recommend: ['高腰 A 字裙', '直筒裤'],
  avoid: ['低腰裤', '臀部复杂装饰'],
  reason: 'A 型轮廓适合弱化下半身横向感，并用高腰线拉长比例。',
  priority: 96
}
```

## AI 的位置

推荐判断应先由知识库完成，AI 只负责把结构化结果润色成更自然的表达。

不建议让 AI 直接自由判断穿搭，因为这样会带来：

- 结果不可解释。
- 同一用户多次测评不稳定。
- 容易生成与专业规则冲突的建议。
- 难以控制敏感表达和身材评价。

## 后续扩展

- 图片辅助识别：识别脸型、肤色、体型，仅作为用户确认前的辅助。
- 衣橱管理：把用户已有单品映射到推荐路线。
- 商品导购：把建议单品映射到真实商品。
- 真人顾问复核：高级报告由形象顾问二次确认。
